For my PROG5121 POE, I combined Part 2 and Part 3 into one integrated QuickChat application. Instead of creating separate programs, I extended the Part 2 messaging system to include all the additional requirements from Part 3. After a user successfully registers and logs in, they can access the QuickChat system where they can send, store, or disregard messages. I also added message IDs, message hashes, recipient validation, message length validation, and message storage functionality. Combining the two parts allowed the application to flow naturally from registration and login directly into the messaging system, making it more user-friendly and realistic.

I also combined the Part 2 and Part 3 test packages to make testing more organised and easier to manage. Rather than creating separate testing sections, I grouped all JUnit and automated tests together while keeping the main classes separate. The tests cover username validation, password complexity, cellphone number validation, login functionality, message validation, message searching, message reports, and other messaging features. This approach reduced duplication, improved organisation, and made it easier to ensure that all parts of the system worked together correctly. Overall, combining Parts 2 and 3 helped me better understand object-oriented programming, class integration, testing, debugging, validation, and file handling in Java.
